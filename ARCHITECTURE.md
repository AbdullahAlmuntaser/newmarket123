# 🏗️ SystemMarket ERP - بنية النظام المعمارية

## نظرة عامة
نظام ERP متكامل مبني باستخدام Flutter و Drift (SQLite) مع تطبيق مبادئ Clean Architecture.

## 📐 البنية المعمارية

### 1. Clean Architecture Layers

```
┌─────────────────────────────────────┐
│     Presentation Layer              │
│  (UI, Widgets, Screens, BLoCs)      │
├─────────────────────────────────────┤
│     Domain Layer                    │
│  (Entities, Use Cases, Repositories)│
├─────────────────────────────────────┤
│     Data Layer                      │
│  (Repositories Impl, Services, DB)  │
└─────────────────────────────────────┘
```

### 2. هيكل المشروع

```
lib/
├── core/                    # النواة الأساسية
│   ├── services/           # خدمات الأعمال
│   ├── datasources/        # مصادر البيانات
│   ├── utils/              # أدوات مساعدة
│   └── constants/          # ثوابت النظام
├── data/                   # طبقة البيانات
│   ├── models/            # نماذج البيانات
│   └── repositories/      # تنفيذ المستودعات
├── domain/                 # طبقة المجال
│   ├── entities/          # الكيانات الأساسية
│   ├── repositories/      # واجهات المستودعات
│   └── usecases/          # حالات الاستخدام
├── presentation/           # طبقة العرض
│   ├── features/          # الميزات مقسمة
│   ├── widgets/           # مكونات واجهة مشتركة
│   └── blocs/             # إدارة الحالة
└── injection_container.dart # حقن التبعيات
```

## 🔄 تدفق البيانات

```
1. UI (SalesInvoicePage)
   ↓
2. BLoC (SalesBloc)
   ↓
3. Service (SalesService)
   ↓

   ↓

   ↓
6. Stream → تحديث الواجهة
```

## 🔐 الأمان والصلاحيات

### نظام الصلاحيات (RBAC)
- Admin: صلاحيات كاملة
- Manager: عمليات يومية + تقارير
- Cashier: مبيعات فقط
- StoreKeeper: مخزون فقط

### الحماية المطبقة:
- تحقق من الصلاحيات قبل كل عملية حساسة
- Audit Logs لجميع العمليات
- منع المخزون السلبي
- توازن القيود المحاسبية

## 📊 قاعدة البيانات

### الجداول الرئيسية:
- `products` - الأصناف
- `partners` - العملاء/الموردين
- `sales` + `sale_items` - المبيعات
- `purchase_orders` + `purchase_items` - المشتريات
- `stock_movements` - حركات المخزون
- `gl_entries` + `gl_lines` - القيود المحاسبية
- `accounts` - شجرة الحسابات

### الفهارس (Indexes):
- `products_sku_idx` - بحث سريع بالـ SKU
- `products_barcode_idx` - مسح الباركود
- `sale_items_sale_id_idx` - استعلامات الفواتير
- `gl_lines_account_id_idx` - تقارير الحسابات
