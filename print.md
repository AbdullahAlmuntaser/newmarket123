أنت مهندس Flutter خبير في أنظمة ERP (POS + Inventory + Accounting) ومتخصص في تحليل المشاريع وتحسينها.

🎯 المطلوب:
تحليل المشروع الحالي بشكل كامل، ثم:

- إذا الأنظمة موجودة → تحسينها
- إذا غير موجودة → بناؤها من الصفر

⚠️ مهم:
لا تفترض أي شيء — حلل الكود فعلياً أولاً

━━━━━━━━━━━━━━━━━━━━━
🧠 أولاً: تحليل الواجهات (POS + Purchases)

افحص المشروع وابحث عن:

- PosScreen / SalesScreen
- PurchaseScreen

📌 تحقق من:

1. هل يتم استدعاؤها داخل التطبيق (Navigation)؟
2. هل مربوطة بـ:
   - SalesService
      - PurchaseService
      3. هل تحتوي على:
         - إضافة منتجات
            - سلة (Cart)
               - حساب الإجمالي
                  - حفظ الفاتورة

                  📌 صنّف الحالة:

                  - ✅ موجودة وتعمل
                  - ⚠️ موجودة لكنها ناقصة
                  - ❌ غير موجودة

                  ━━━━━━━━━━━━━━━━━━━━━
                  📦 ثانياً: تقرير الواجهات

                  حدد:

                  - هل POS يعمل فعلياً أم مجرد UI
                  - هل المشتريات تعمل فعلياً
                  - ما هي الأجزاء الناقصة بالتحديد

                  ━━━━━━━━━━━━━━━━━━━━━
                  🧠 ثالثاً: تحليل نظام الوحدات (Units)

                  تحقق من:

                  - وجود Units Table
                  - وجود ProductUnits
                  - وجود conversion_factor
                  - هل يتم التحويل داخلياً أم يدوياً

                  📌 صنّف:

                  - ✅ كامل
                  - ⚠️ ناقص
                  - ❌ غير موجود

                  ━━━━━━━━━━━━━━━━━━━━━
                  🔧 رابعاً: تحسين أو بناء نظام الوحدات

                  إذا موجود:

                  - منع الإدخال اليدوي للتحويل
                  - استخدام conversion_factor من DB
                  - Dropdown للوحدة + حقل كمية واحد

                  إذا غير موجود:

                  - أنشئ:
                    - Units
                      - ProductUnits
                        - ProductPrices

                        ━━━━━━━━━━━━━━━━━━━━━
                        💰 خامساً: تحليل نظام التسعير

                        تحقق من:

                        - هل يوجد سعر لكل وحدة؟
                        - هل يوجد جملة / تجزئة؟
                        - هل يتم تحديد السعر تلقائياً؟

                        ━━━━━━━━━━━━━━━━━━━━━
                        🔧 سادساً: Pricing Engine

                        قم ببناء أو تحسين:

                        - دعم:
                          - retail
                            - wholesale
                            - سعر لكل وحدة
                            - اختيار السعر حسب الكمية

                            ━━━━━━━━━━━━━━━━━━━━━
                            💸 سابعاً: الخصومات

                            - خصم على مستوى الصنف
                            - خصم على الفاتورة
                            - نسبة أو مبلغ ثابت

                            ━━━━━━━━━━━━━━━━━━━━━
                            🧾 ثامناً: الضرائب

                            - دعم ضريبة %
                            - حسابها بعد الخصم

                            formula:

                            subtotal = sum(items)
                            discount = ...
                            tax = (subtotal - discount) * taxRate
                            total = subtotal - discount + tax

                            ━━━━━━━━━━━━━━━━━━━━━
                            🧠 تاسعاً: منطق الحساب

                            لكل عنصر:

                            quantity × conversion_factor × unit_price

                            ━━━━━━━━━━━━━━━━━━━━━
                            🔗 عاشراً: الربط مع النظام

                            تأكد من:

                            - البيع → ينقص المخزون
                            - الشراء → يزيد المخزون
                            - كل عملية → قيد محاسبي

                            ━━━━━━━━━━━━━━━━━━━━━
                            📊 الحادي عشر: التحقق النهائي

                            - هل النظام يعمل فعلياً؟
                            - هل الحسابات صحيحة؟
                            - هل يوجد إدخال يدوي غير ضروري؟

                            ━━━━━━━━━━━━━━━━━━━━━
